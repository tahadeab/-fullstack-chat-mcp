# Contributing to Fullstack Chat MCP

Thank you for your interest in contributing! This document provides guidelines for contributing to the project.

## 🎯 Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Focus on what's best for the community

## 🚀 Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/fullstack-chat-mcp.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Set up the development environment (see README.md)

## 📝 Development Guidelines

### Code Style
- Use meaningful variable and function names
- Keep functions small and focused
- Add comments for complex logic
- Follow existing code conventions

### Commits
- Write clear, descriptive commit messages
- Use present tense ("Add feature" not "Added feature")
- Reference issues when applicable

### Pull Requests
1. Ensure your code passes all CI checks
2. Update documentation if needed
3. Describe your changes clearly
4. Link related issues

## 🧪 Testing

Before submitting a PR:
- Test your changes locally
- Ensure existing tests pass
- Add tests for new features

## 📦 Project Structure

```
chat-app/
├── backend/          # Node.js + Express API
├── frontend/         # React + Vite UI
├── mcp-server/       # MCP AI integration
└── .github/          # CI/CD workflows
```

## 🐛 Reporting Issues

When reporting bugs, include:
- Steps to reproduce
- Expected vs actual behavior
- Environment details (OS, Node version, etc.)
- Screenshots if applicable

## 💡 Feature Requests

Feature requests are welcome! Please include:
- Clear description of the feature
- Use case or problem it solves
- Any implementation ideas

## 🔒 Security

For security issues, please contact the maintainer directly instead of opening a public issue.
